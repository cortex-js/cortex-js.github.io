Collections are used to represent data structures.

A frequently used collection is the List which is used to represent an ordered sequence of elements.

["List", 42, 3.14, "x", "y"]
\lbrack 42, 3.14, x, y \rbrack
\[ \lbrack 42, 3.14, x, y \rbrack \]

Lists can be used to represent vectors.

["List", 1, 2, 3]
\lbrack 1, 2, 3 \rbrack
\[ \lbrack 1, 2, 3 \rbrack \]

A list of lists can be used to represent a matrix.

["List", ["List", 1, 2, 3], ["List", 4, 5, 6], ["List", 7, 8, 9]]
\lbrack \lbrack 1, 2, 3 \rbrack, \lbrack 4, 5, 6 \rbrack, \lbrack 7, 8, 9 \rbrack \rbrack
\[ \lbrack \lbrack 1, 2, 3 \rbrack, \lbrack 4, 5, 6 \rbrack, \lbrack 7, 8, 9 \rbrack \rbrack \]

Lists of lists can also be represented using a ; separator:

\lbrack 1, 2, 3 ; 4, 5, 6 ; 7, 8, 9 \rbrack
\[ \lbrack 1, 2, 3 ; 4, 5, 6 ; 7, 8, 9 \rbrack \]

And matrixes can be represented using LaTeX environments:

\begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \\ 7 & 8 & 9 \end{pmatrix}
\[ \begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \\ 7 & 8 & 9 \end{pmatrix} \]

Another common collection is the Range which is used to represent a sequence of numbers.

["Range", 1, 10]
\lbrack 1..10 \rbrack
\[ \lbrack 1..10 \rbrack \]

Collection operations such as IsEmpty, Take, IndexOf can be applied to any collection types.

["Take", ["List", 2, 5, 7], 2]
// -> 5
\lbrack 2, 5, 7 \rbrack_{2}
\[ \lbrack 2, 5, 7 \rbrack_{2} \]
["Take", ["Range", 2, 10], 5]
// -> 7
(2..10)_5
\[ (2..10)_5 \]

Collection Types

List

["List", expr-1, expr-2]

An ordered collection of elements.

Use to represent a data structure, unlike Delimiter which is just a visual styling.

MathJSON LaTeX
["List", "x", "y", "7", "11"] \( \lbrack x, y, 7, 11\rbrack \)
["List", "x", "Nothing", "y"] \( \lbrack x,y\rbrack \)

Range

["Range", lower, upper, step]

["Range", lower, upper]

["Range", upper]

A sequence of numbers.

If the step is not specified, it is assumed to be 1.

If there is a single argument, it is assumed to be the upper bound, and the lower bound is assumed to be 1.

Linspace

["Linspace", lower, upper, count]

["Linspace", lower, upper]

["Linspace", upper]

A sequence of numbers. Similar to Range but the count is specified instead of the step.

If the count is not specified, it is assumed to be 50.

If there is a single argument, it is assumed to be the upper bound, and the lower bound is assumed to be 1.

Set

["Set", expr-1, expr-2]

An unordered collection of unique elements.

MathJSON LaTeX
["Set", "x", "y"] \( \lbrace x, y\rbrace \)

Collection Operations

Length

["Length", collection]

Returns the number of elements in the collection.

Note this can also be used to get the length of a string.

["Length", ["List", 5, 2, 10, 18]]
// -> 4

["Length", "Hello"]
// -> 5

IsEmpty

["IsEmpty", collection]

Returns the symbol True if the collection is empty.

["IsEmpty", ["List", 5, 2, 10, 18]]
// -> False

["IsEmpty", ["List"]]
// -> True

["IsEmpty", "x"]
// -> True


["IsEmpty", "Hello"]
// -> 5

Take

["Take", collection, index]{.signature} ["Take", collection, index1, index2]

Returns a list of the elements at the specified indexes.

["Take", ["List", 5, 2, 10, 18], 2]
// -> ["List", 10]

["Take", ["List", 5, 2, 10, 18], -2, 1]
// -> ["List", 10, 5]

["Take", collection, range]

Returns a list of the elements at the specified ranges.

Each range is specified as a tuple with the following elements:

  • start: the starting index of the range
  • end: the ending index of the range
  • step: the step of the range (1 if not omitted)
["Take", ["List", 5, 2, 10, 18], ["Tuple", 2, 3]]
// -> ["List", 10, 18]

["Take", ["List", 5, 2, 10, 18], ["Tuple", 1, -1, 2]]
// -> ["List", 5, 10]

The elements are returned in the order in which theyre specified. Using negative indexes (or ranges) reverses the order of the elements.

["Take", ["List", 5, 2, 10, 18], ["Tuple", -1, 1]]
// -> ["List", 18, 10, 2, 5]

["Take", "'desserts'", ["Tuple", -1, 1]]
// -> "'stressed'"

Drop

["Drop", collection, index]

Drop is the opposite of Take. It returns a list of the elements that are not at the specified indexes.

["Drop", ["List", 5, 2, 10, 18], 2]
// -> ["List", 5, 10, 18]

["Drop", ["List", 5, 2, 10, 18], -2, 1]
// -> ["List", 2, 18]

["Drop", collection, range]

It also accepts a list of ranges.

["Drop", ["List", 5, 2, 10, 18], ["Tuple", 2, 3]]
// -> ["List", 5, 2]
["Drop", ["List", 5, 2, 10, 18], ["Tuple", 1, -1, 2]]

// -> ["List", 2, 18]

First

["First", collection]

Return the first element of the collection.

Its equivalent to ["Take", _collection_, 1].

["First", ["List", 5, 2, 10, 18]]
// -> 5

["First", ["Tuple", "x", "y"]]
// -> "x"

Second

["Second", collection]

Return the first element of the collection.

Its equivalent to ["Take", _collection_, 2].

["Second", ["Tuple", "x", "y"]]
// -> "y"

Last

["Last", collection]

Return the last element of the collection.

Its equivalent to ["Take", _collection_, -1].

["Last", ["List", 5, 2, 10, 18]]
// -> 18

Rest

["Rest", collection]

Return everything but the first element of the collection.

Its equivalent to ["Drop", _collection_, 1].

["Rest", ["List", 5, 2, 10, 18]]
// -> ["List", 2, 10, 18]

Most

["Most", collection]

Return everything but the last element of the collection.

Its equivalent to ["Drop", _collection_, -1].

["Most", ["List", 5, 2, 10, 18]]
// -> ["List", 5, 2, 10]

Reverse

["Reverse", collection]

Return the collection in reverse order.

Its equivalent to ["Take", _collection_, ["Tuple", -1, 1]].

["Reverse", ["List", 5, 2, 10, 18]]
// -> ["List", 18, 10, 2, 5]
